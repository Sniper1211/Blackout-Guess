# 优化游戏布局：响应式双栏设计

为了解决长篇诗词导致输入框被顶出屏幕的问题，我们将重构页面的核心布局，在桌面端采用横向排列方式，确保输入区始终可见。

## 1. 结构重构 (index.html)
- 在 `game-container` 内部引入一个新的布局容器 `.game-main-layout`。
- 将页面划分为两个主要区域：
  - **展示区 (`.game-primary-area`)**：包含 `text-display`（诗词文本）。
  - **控制区 (`.game-sidebar`)**：包含 `stats-container`（统计）、`input-section`（输入框）、`message`（提示消息）和 `guessed-letters`（已猜字符）。

## 2. 样式优化 (styles.css)
- **桌面端 (屏幕宽度 > 992px)**:
  - `.game-container`: 将 `max-width` 从 800px 增加到 1100px，提供更充裕的横向空间。
  - `.game-main-layout`: 使用 `display: flex; align-items: flex-start; gap: 30px;` 实现左右布局。
  - **展示区**: `flex: 1;` 占据主要空间。
  - **控制区**: 
    - 固定宽度（约 320px）。
    - 使用 `position: sticky; top: 100px;` 确保在长诗滚动时，输入框和统计信息始终停留在屏幕内。
  - 调整 `stats-container` 和 `input-group` 的内部样式，使其在侧边栏中表现更紧凑。

- **移动端 (默认)**:
  - 维持纵向堆叠布局 (`flex-direction: column`)。
  - 保持原有的简洁风格，确保小屏操作体验。

## 3. 预期效果
- **短诗**: 左右平衡，界面精致。
- **长诗**: 左侧诗词可垂直滚动或自然延伸，右侧输入区固定在屏幕右侧，玩家无需滚动即可随时输入猜测，极大提升操作连贯性。
