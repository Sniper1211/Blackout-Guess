## 问题分析

基于用户的需求（今天是2026年1月28日，希望从1月21日开始每天有且只有一个诗词题目），我发现了以下问题：

### 1. 日历样式问题
- `.calendar-date.other-month` 样式中的 `color: #bbb` 在浅色背景下对比度不足
- 缺少暗色模式下的对应样式

### 2. 数据库配置问题
- 需要确保数据库中有从1月21日到1月28日的题目数据
- 需要启用每日一题模式（设置 `daily_mode_enabled` 为 `true`）

## 解决方案

### 第一步：修复日历样式
1. 修改 `styles.css` 中的 `.calendar-date.other-month` 样式，增加对比度
2. 添加暗色模式下的 `.dark-theme .calendar-date.other-month` 样式

### 第二步：配置每日一题系统
1. 检查并更新 `app_settings` 表，将 `daily_mode_enabled` 设置为 `true`
2. 确保数据库中有从2026-01-21到2026-01-28的题目数据，每个日期对应一个题目

### 第三步：测试功能
1. 测试从1月21日到1月28日的题目加载
2. 验证日历点击功能
3. 确保每天有且只有一个题目

## 具体修改内容

### 样式修复：
- 将 `.calendar-date.other-month` 的 `color: #bbb` 改为 `color: #888`
- 添加 `.dark-theme .calendar-date.other-month { color: #aaa; }`

### 数据库配置：
- 执行SQL：`UPDATE public.app_settings SET daily_mode_enabled = true WHERE id = 'global';`
- 确保每个日期（2026-01-21到2026-01-28）都有对应的题目，状态为 `published`

## 预期结果
1. 日历中本月剩余几天的日期清晰可见
2. 从1月21日开始，每天有且只有一个题目
3. 点击日历上的日期可以加载对应的题目
4. 刷新页面题目保持不变（同一天内）
