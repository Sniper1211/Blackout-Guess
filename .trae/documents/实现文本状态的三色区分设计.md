# 文本状态颜色区分设计方案

为了满足您对三种文字状态（用户猜中、系统提示、自动揭示）的视觉区分需求，我将引入一套新的状态管理机制和配色方案。

## 1. 视觉设计方案 (配色)

我们将沿用“新中式雅致”的主题，为三种状态分配具有语义的颜色：

1.  **猜测正确 (User Guessed)**: **靛蓝 (Indigo)**
    *   颜色：`#40739e` (现有主色)
    *   含义：代表用户的主动墨迹，最显著、最令人满意的颜色。
    *   样式：深蓝底白字。

2.  **系统提示 (Hinted)**: **琥珀/藤黄 (Amber/Ochre)**
    *   颜色：`#e67e22` 或 `#d35400`
    *   含义：代表“批注”或“金印”，警示与辅助并存。既醒目又与用户的墨迹区分开。
    *   样式：橙黄底白字。

3.  **自动揭示 (Auto Revealed)**: **淡墨/青灰 (Faded Ink/Grey)**
    *   颜色：`#95a5a6` 或 `#7f8c8d`
    *   含义：代表“补全”的文字，色彩饱和度低，不喧宾夺主，表示这是游戏结束后的补充信息。
    *   样式：灰底白字。

## 2. 技术实现步骤

### 第一步：升级游戏引擎 (`js/GameEngine.js`)
目前的数据结构只区分了“隐藏”和“显示”，以及是否“用户猜测”。我们需要增加更精确的状态追踪。
*   在 `hiddenText` 的数据项中引入 `revealType` 字段，可能的值为：
    *   `'user'`: 用户猜中
    *   `'hint'`: 使用提示揭示
    *   `'auto'`: 游戏胜利/放弃后自动补全
*   修改 `guessLetter`、`useHint` 和 `revealAllText` 方法，在改变状态时记录对应的 `revealType`。

### 第二步：更新 UI 渲染 (`js/UIManager.js`)
*   修改 `updateTextDisplay` 方法。
*   根据 `item.revealType` 应用不同的 CSS 类名：
    *   `.guessed-by-user` (保持不变)
    *   `.revealed-by-hint` (新增)
    *   `.revealed-auto` (新增)

### 第三步：添加样式 (`styles.css`)
*   定义 `.revealed-by-hint` 和 `.revealed-auto` 的样式。
*   确保所有新状态都严格继承 `40px * 40px` 的尺寸规则，防止布局跳动。

## 3. 预期效果
*   **猜对的字**：蓝色方块，翻转入场。
*   **提示的字**：橙色方块，区分度高。
*   **最后补全的字**：灰色方块，安静地填充剩余空白。
